MISA：AIエージェント研修 レポート

研修概要

本研修は、MISAにおけるDX人材育成の一環として、AIエージェントの基礎から実践的な活用までを学ぶことを目的としています。コマンドライン操作やGitHubを用いた基本的な開発環境の構築から始まり、AIを活用したアプリケーションの作成と公開、さらには最新のModel Context Protocol (MCP) の理解と利用を通じて、AIエージェントを効果的に業務へ導入するための知識とスキルを習得します。

各回は段階的に内容が深まる構成となっており、実践的な演習を通じて、AIエージェント開発に必要な一連のプロセスを体験できます。本レポートは、研修内容の要点をまとめ、社内での情報共有を促進することを目的としています。

# コマンドプロンプトの利点と使い方
# コマンドプロンプトは、キーボード入力だけでPCを操作できる強力なツールです。マウス操作よりも高速で、繰り返し作業の自動化にも向いています。AIエージェント開発では、ツールの実行やファイルの管理など、様々な場面で活用します。

# コマンドプロンプトの起動方法:
# Windowsキー + R を押し、「cmd」と入力してEnterキーを押してください。

# 基本コマンド一覧
# --------------------------------------------------------------------------------
# 1. 現在の作業ディレクトリを表示 (今、自分がPCのどこにいるかを確認)
#    コマンド: cd
#    例: cd
# --------------------------------------------------------------------------------
# 2. ディレクトリを移動する (作業場所を変更する)
#    コマンド: cd [移動したいディレクトリのパス]
#    例: cd C:\Users\YourUser\Documents
# --------------------------------------------------------------------------------
# 3. ファイルやフォルダの一覧を表示する (現在のディレクトリにあるものを見る)
#    コマンド: dir
#    例: dir
# --------------------------------------------------------------------------------
# 4. 新しいフォルダを作成する
#    コマンド: mkdir [作成したいフォルダ名]
#    例: mkdir MyProject
# --------------------------------------------------------------------------------
# 5. テキストファイルを作成する
#    コマンド: echo [書き込みたい内容] > [ファイル名]
#    例: echo Hello World! > hello.txt
# --------------------------------------------------------------------------------
# 6. テキストファイルの内容を表示する
#    コマンド: type [ファイル名]
#    例: type hello.txt
# --------------------------------------------------------------------------------

# GitHubのセットアップと基本操作
# GitHubは、あなたの作ったプログラムのコードやファイルをインターネット上で管理・共有するための「開発者のためのSNS」のようなものです。チームでの共同開発や、自分のプロジェクトの変更履歴を自動で記録するのに非常に役立ちます。

# 1. GitHubアカウントの作成とログイン
#    まだアカウントがない場合:
#    GitHub公式サイト (https://github.com/) にアクセスし、「Sign up」ボタンから画面の指示に従ってアカウントを作成してください。
#    アカウント作成後:
#    GitHub公式サイト (https://github.com/) にアクセスし、「Sign in」から作成したアカウントでログインしてください。

# 2. Gitのインストール
#    Gitは、ファイルの変更履歴を管理するためのツールです。GitHubと連携して使います。あなたのPCにGitをインストールしましょう。
#    Git for Windows公式サイト (https://gitforwindows.org/) にアクセスし、インストーラーをダウンロード・実行してください。インストールは基本的に「Next」を押し続けるだけでOKです。

# 3. Gitの初期設定 (コマンドプロンプトで実行)
#    Gitをインストールしたら、まずあなたのPCでGitを使うための初期設定を行います。コマンドプロンプトを開いて、以下のコマンドを一つずつ実行してください。
#    ここで設定するユーザー名とメールアドレスは、あなたがGitHubにアップロードする変更履歴に「誰が変更したか」を記録するために使われます。
#    git config --global user.name "あなたのGitHubユーザー名"  (例: "your-username")
#    git config --global user.email "あなたのGitHub登録メールアドレス" (例: "your.email@example.com")

# 4. 新しいリポジトリの作成 (GitHub上)
#    GitHubにログインした状態で、新しいプロジェクトの「保管場所（リポジトリ）」を作成します。
#    a. GitHubの画面右上にある「+」アイコンをクリックし、「New repository」を選択します。
#    b. 「Repository name」にプロジェクトの名前（例: `my-first-repo`）を入力します。
#    c. 「Public」（公開）または「Private」（非公開）を選択します。最初は「Public」で問題ありません。
#    d. **重要:** 「Add a README file」や「Add .gitignore」などのチェックボックスは**すべてチェックを外してください**。これらは後でローカルで作成します。
#    e. 「Create repository」ボタンをクリックします。
#    f. 作成されたリポジトリのページに表示される「...or create a new repository on the command line」の下にあるURL（例: `https://github.com/あなたのユーザー名/あなたのリポジトリ名.git`）をコピーしておきましょう。

# 5. ローカルリポジトリの作成とGitHubへの連携 (コマンドプロンプトで実行)
#    あなたのPC上にプロジェクト用のフォルダを作成し、そのフォルダをGitで管理できるようにします。そして、先ほどGitHubで作成したリポジトリと連携させます。
#    a. コマンドプロンプトで、プロジェクトを作成したい場所（例: `C:\Users\YourUser\Documents`）に移動します。
#    b. 新しいプロジェクトフォルダを作成し、その中に移動します。
#       mkdir MyNewProject
#       cd MyNewProject
#    c. このフォルダをGitで管理するための初期化を行います。
#       git init
#    d. ローカルのフォルダとGitHub上のリポジトリを連携させます。「origin」はGitHub上のリポジトリを指す「ニックネーム」のようなものです。
#       git remote add origin コピーしたリポジトリのURL (例: `https://github.com/あなたのユーザー名/あなたのリポジトリ名.git`)

# 6. ファイルの追加とコミット (変更履歴の記録)
#    プロジェクトフォルダにファイルを作成・編集した後、その変更をGitに記録します。Gitでは、変更を「コミット」という単位で記録します。
#    a. まず、簡単なファイルを作成してみましょう。
#       echo # My First Project > README.md
#    b. 作成したファイルをGitの管理対象に追加します。
#       git add README.md
#    c. 変更内容を記録します。`-m`の後に、その変更がどんな内容だったかを分かりやすく記述します。
#       git commit -m "Initial commit: Add README file"

# 7. GitHubへのプッシュ (変更をGitHubにアップロード)
#    ローカルで記録した変更履歴を、GitHub上のリポジトリにアップロードします。これにより、他の人と共有したり、PCが壊れても履歴が残るようになります。
#    a. 最初のプッシュでは、`-u origin main`というオプションをつけます。これは「今後、このローカルの`main`ブランチはGitHubの`origin`という名前のリポジトリの`main`ブランチと連携するよ」という設定を意味します。

演習：Gitの基本操作

ゴール: GitHubとローカルをつなげて、 add → commit → push の流れを体験する。

① GitHubでリポジトリを作成

  1. ブラウザでGitHubにログイン
  2. 「New repository」からリポジトリを作成
    • 名前: practice-git
    • READMEは作らないでOK
  3. 作成後に表示されるURL（HTTPS）をコピー

② ローカルで練習用フォルダを作る

mkdir practice-git
cd practice-git

③ テキストファイルを作る

echo "はじめてのGit練習です。" > hello.txt

④ Gitを初期化してリモートと接続

git init
git branch -M main
git remote add origin https://github.com/<ユーザー名>/practice-git.git

⑤ ファイルを登録して送る

git add .
git commit -m "最初のコミット"
git push -u origin main

GitHubに hello.txt がアップロードされているか確認！

⑥ ファイルを変更して再度push

echo "2回目の変更です。" >> hello.txt
git add .
git commit -m "hello.txtを更新"
git push

GitHubの hello.txt に変更が反映されているか確認！

# ステップA：Live Serverのインストール（初回のみ）

#
